# Bitcoin Wiki UI Repair Design

**Goal:** Quartz 기본 구조를 유지한 채 데스크톱에서 `왼쪽 탐색기 / 가운데 본문 / 오른쪽 그래프` 3열 레이아웃을 안정적으로 복구하고, 그래프를 패널 안에 꽉 차게 보이도록 만든다.

## 문제 요약

- 이전 시도는 루트 레이아웃을 과하게 덮어쓰면서 Quartz 기본 반응형 구조와 충돌했다.
- 그 결과 일부 페이지가 모바일처럼 1열로 무너지거나, 그래프 패널과 실제 캔버스 크기가 어긋났다.
- 홈과 일반 문서 페이지에서 그래프 동작도 분리되지 않아 탐색 경험이 불명확했다.

## 승인된 구조

- 왼쪽: 얇은 탐색기
  - `PageTitle`
  - `Search`
  - `Darkmode`
  - `ReaderMode`
  - `Explorer`
- 가운데: 본문 최우선
  - 제목과 본문 폭을 가장 크게 둔다.
- 오른쪽: 대형 그래프 패널
  - 그래프를 최상단에 두고, 그 아래에 `TOC`와 `Backlinks`를 둔다.

## 그래프 동작

- 홈 페이지는 전체 그래프를 보여 준다.
- 일반 문서 페이지는 해당 문서 기준 로컬 그래프만 보여 준다.
- 그래프 패널은 명시적 높이를 갖고, 내부 캔버스도 컨테이너 크기를 따라 다시 계산해야 한다.

## 비주얼 방향

- 과한 네온이 아니라 절제된 사이퍼펑크/비트코인 맥시 톤
- 흑갈색 바탕, 브론즈/비트코인 오렌지 강조, 얇은 그리드 질감
- 본문은 가장 읽기 쉬운 영역으로 유지
- 그래프 패널은 가장 강한 시각 포인트로 만든다

## 구현 안전선

- `quartz-overrides`와 그래프 스크립트 오버라이드 범위만 수정한다.
- Quartz 기본 3영역 grid는 유지하고, 컬럼 비율과 패널 스타일만 조정한다.
- UI 변경은 `content/` 문서 검증 작업과 분리한다.

## 검증 기준

- 로컬 빌드 성공
- 홈 페이지에서 3열 레이아웃 유지
- 일반 문서 페이지에서 3열 레이아웃 유지
- 홈은 글로벌 그래프, 일반 문서는 로컬 그래프 동작
- 그래프 캔버스가 패널 박스를 꽉 채움
- 모바일에서는 자연스럽게 1열로 접힘
