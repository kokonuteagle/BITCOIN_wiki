# 비트코인 모노리스 · Bitcoin Monolith Quartz Vault

이 저장소는 **비트코인 단일 주제 위키**다. 목표는 세 가지를 동시에 만족시키는 것이다.

1. **주제별 폴더 + 엔터티별 Markdown 파일**
2. **Obsidian에서 관리 가능한 로컬 볼트**
3. **Quartz 4로 GitHub Pages에 퍼블리시되는 세련된 위키**

현재 v3 기준으로 볼트에는 **131개 Markdown 문서**와 **19개 Mermaid 다이어그램**이 들어 있다. 평균 본문 길이는 약 **526.4단어**, 중앙값은 **515단어** 수준이다. 내용은 통화철학, 프로토콜, 업그레이드, 셀프커스터디, 채굴, 라이트닝, 프라이버시, 역사 논쟁, 도서관, 인물 지도를 한 덩어리로 엮는다.

핵심 설계 원칙은 다음과 같다.

- 문서는 모두 `content/` 아래의 Markdown 파일이다.
- 링크는 Obsidian 위키링크 문법으로 작성된다.
- Quartz가 백링크, 그래프 뷰, 검색, 폴더 탐색을 웹에서 제공한다.
- 각 문서는 단순 정의가 아니라 **개념적 위치, 역사, 논쟁, 실무 적용, 다음 읽기**까지 포함한다.
- 최대한 **비트코인 온리 / 셀프커스터디 / 검열저항 / 낮은 시간선호** 관점에서 서술하되, 독단적 구호가 아니라 기술적·경제학적 논증을 우선한다.


## v3 보강 사항

- 기존 표기를 전반적으로 `셀프커스터디`로 정리
- 모든 콘텐츠 문서를 **개별 기준 최소 2배 이상** 확장
- 인물/도서 문서를 카드형 요약에서 **입체적 해설 노드**로 증보
- 검증 보고서에 링크 무결성, 그래프 도달성, 평균 길이 통계 추가

## 로컬에서 Obsidian으로 쓰기

- 저장소 전체를 받아도 되지만, 실질적으로는 `content/` 폴더를 Obsidian vault로 열면 된다.
- Quartz 웹 전용 기능인 백링크와 그래프는 웹에서 제공되지만, 링크 구조는 Obsidian에서도 그대로 살아 있다.
- 문서를 추가할 때는 되도록 `허브 -> 개념 -> 인물/도서 -> 가이드/논쟁` 흐름으로 링크를 걸어 주면 그래프가 안정된다.

## 로컬에서 미리 보기

준비물:
- Node.js 22+
- Git
- 인터넷 연결 1회 이상 필요

명령어:

```bash
npm run dev
```

처음 실행 시 다음이 자동으로 수행된다.

1. Quartz v4 clone
2. Quartz 의존성 설치
3. `content/`와 `quartz-overrides/`를 엔진으로 동기화
4. 로컬 개발 서버 실행

정적 빌드:

```bash
npm run build
```

## GitHub Pages 배포

1. 이 폴더 전체를 GitHub 저장소에 push
2. 저장소 **Settings -> Pages** 로 이동
3. **Source** 를 **GitHub Actions** 로 변경
4. `main` 브랜치에 push 하면 자동 배포

## 디자인 포인트

- 좌측: 탐색기 + 검색 + 다크모드 + 리더모드
- 본문: 백과사전형 문서와 긴 에세이를 모두 소화하는 폭
- 우측: 글로벌 그래프 + 목차 + 백링크
- 색감: 비트코인 오렌지의 채도를 낮추고, 종이색과 흑연색을 섞은 세련된 대비
- Mermaid 다이어그램: 개념 관계, 프로토콜 흐름, 역사적 분기점, 실무 절차를 시각화
